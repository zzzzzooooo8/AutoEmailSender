# 开发模式

## 运行前端

```
cd frontend

npm install

npm run dev
```

## 运行后端

```
cd backend

uv run uvicorn main:app --reload
```

## 第一次使用建议顺序

1. 先启动后端与前端，确认页面可以正常打开。
2. 进入个人页，先创建发件身份并完成 SMTP 测试。
3. 继续在个人页配置模型、上传材料，并准备默认模板。
4. 进入导师管理页，导入或录入第一批导师。
5. 回到首页先筛选并选择导师，再进入创建任务页创建第一批任务。
6. 进入工作区生成匹配与草稿，人工确认后再发送。

首次跑通整条流程时，建议先保持顶部发送模式为 `dry_run`。
