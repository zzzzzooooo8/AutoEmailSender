# 开发模式

## 运行前端

```
cd frontend

npm install

npm run dev
```

## 运行后端

```
cd backend

uv run uvicorn main:app --reload
```

## 第一次使用建议顺序

1. 先启动后端与前端，确认页面可以正常打开。
2. 进入个人页，先完成发件身份配置，把 SMTP 和 IMAP 一起确认好。
3. 继续在个人页准备材料与默认模板。
4. 在个人页配置模型，并完成可用性测试。
5. 进入导师管理页，导入或录入第一批导师。
6. 回到首页先筛选并选择导师，再进入创建任务页创建第一批任务。
7. 进入工作区生成匹配与草稿，人工确认后再发送。

首次跑通整条流程时，建议先保持顶部发送模式为 `dry_run`。
