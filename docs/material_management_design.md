# 统一材料管理设计

## 1. 设计目标
- 把原来的“主简历 + 附件”拆分心智收敛成统一材料库。
- 让用户在身份页只维护一套材料，不需要理解底层表结构差异。
- 保留业务语义差异：
  - 默认材料：用于匹配和草稿生成。
  - 随信材料：用于实际发送时作为附件带出。

## 2. 核心概念
### 2.1 材料
- 材料统一存放在 `identity_materials`。
- 每份材料都有轻标签：`resume`、`transcript`、`publication`、`portfolio`、`other`。
- 标签只用于展示和筛选，不做权限或严格流程限制。
- 上传材料时只保存文件与元数据，不在上传阶段强制解析文本。

### 2.2 默认材料
- 每个身份最多只有 1 份默认材料，对应 `identity_profiles.current_primary_material_id`，也允许为空。
- 每个任务也最多只有 1 份默认材料，对应 `email_tasks.primary_material_id`。
- 任务默认材料是快照，不自动跟随身份默认材料变化。
- 只有可提取文本的材料才允许设为默认材料。

### 2.3 随信材料
- 每个任务可选择 0 到多份随信材料，对应 `email_tasks.selected_material_ids`。
- 同一份材料既可以是默认材料，也可以同时被选为随信材料。

## 3. 页面行为
### 3.1 个人页
- 个人页展示统一“材料管理”区。
- 列表字段：
  - 材料名称
  - 标签
  - 原始文件名
  - 大小
  - 上传时间
  - 是否默认材料
- 单条操作：
  - 打开
  - 下载
  - 设为默认材料
  - 删除
- 上传材料时，用户可指定轻标签和展示名。
- 前端不展示任何本地文件路径。

### 3.2 创建任务页
- 显示两块材料选择：
  - 默认材料：单选，默认取身份当前默认材料
  - 随信材料：多选，默认不勾选
- 默认材料不是创建任务的前置条件；留空也可以先创建任务。
- 如果暂时没有默认材料，用户可以之后在工作区手动选择材料，再执行匹配和草稿生成。

### 3.3 工作区
- 展示当前任务默认材料。
- 在任务尚未发送、尚未回信前，允许切换默认材料。
- 切换默认材料后，立即重新生成匹配结果和草稿。
- 工作区里的随信材料是多选，用户可在批准发送前自由调整。

## 4. 删除规则
- 如果材料是某身份当前默认材料，允许直接删除；删除后身份回到“未设默认材料”状态。
- 如果材料被任何未终态任务引用，则禁止删除。
- 如果只被已终态任务引用，可以删除。

## 5. 文件展示规则
- 磁盘上的文件名使用 UUID，不直接暴露给用户。
- 用户只看到 `display_name` 和 `original_filename`。
- “打开”走 `inline` 文件响应，“下载”走附件下载响应。

## 6. 任务语义
- LLM 匹配与草稿生成只在用户手动触发时执行；系统会在那一刻通过 MarkItDown 按需读取 `primary_material_id` 对应默认材料的 Markdown 内容，并把结果缓存到 `extracted_text`。
- 邮件发送只读取 `selected_material_ids` 对应的文件实体。
- 切换默认材料不会自动把它加入随信材料列表；两者仍然分开控制。
