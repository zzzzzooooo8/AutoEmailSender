# 导师管理页设计

## 1. 设计目标
- 把“导师数据管理”和“首页任务流”拆开
- 首页继续做筛选与创建任务
- `/professors` 专门负责导入、编辑、归档和恢复

## 2. 页面结构
### 2.1 顶部标题区
- 页面标题：导师档案工作台
- 展示当前列表数量、当前筛选状态、已选择数量

### 2.2 过滤与工具条
- 三态切换：`正常 / 已删除 / 全部`
- 搜索框：按姓名、邮箱、学校、院系、研究方向过滤
- 右侧操作：
  - 下载模板
  - 导入文件
  - 新增导师
  - 刷新

### 2.3 主列表区
- 以紧凑表格/列表混合布局展示：
  - 姓名
  - 邮箱 / 学校
  - 院系
  - 研究方向
  - 更新时间
  - 操作
- 未归档导师支持勾选；已归档导师不参与批量选择

### 2.4 批量操作条
- 只在存在选中项时出现
- 固定文案是“批量移入回收站”
- 明确说明不会删除历史任务和通信

## 3. 弹窗设计
### 3.1 新增 / 编辑导师弹窗
- 同一套表单复用
- 必填：`姓名 + 邮箱`
- `recent_papers` 用多行输入，一行一篇

### 3.2 导入弹窗
- 左侧：模板下载与格式说明
- 右侧：文件拖拽/选择区与导入结果摘要
- 导入完成后保留结果摘要，并刷新主列表

## 4. 交互规则
- 删除统一表现为“移入回收站”
- 已删除列表只提供“恢复”
- 导入时按邮箱覆盖原记录；如果原记录已归档，会自动恢复
- 首页和创建任务页都不再承担数据管理动作，只保留“去导师管理”的轻入口

## 5. 开发辅助区
- 样例导师导入
- 智能抓取占位入口

这两项保留在导师管理页的次级区域，不再放在首页主工具条。
