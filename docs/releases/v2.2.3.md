# v2.2.3

## 更新内容

### 新增功能

- 新增思考模式模型自适应能力：抓取任务启动前会自动探测当前模型需要的 `extra_body` 参数，并缓存到本地，减少 DeepSeek、MiMo、Qwen、豆包、Kimi、GLM 等 OpenAI 兼容模型在多轮 tool-calling 中的协议错误。
- 新增独立的 `thinking_adaptation_cache` 缓存表，按接口地址和模型名称复用探测结果，多个 LLM Profile 指向同一模型时无需重复学习。
- LLM 运行时新增 `extra_body` 支持，可在普通 Chat Completions 请求和 Responses fallback 中传递思考模式相关参数。

### 体验优化

- 模型连接测试改为更轻量的单轮探活，并兼容思考模型返回空 `content` 但请求可达的情况。
- 抓取任务遇到思考模式协议错误时，会给出更明确的自适应探活提示，方便用户重新测试连接或调整模型配置。
- 抓取 Agent 构建逻辑移除对特定模型的硬编码，统一使用自适应结果决定是否关闭思考模式。

### 问题修复

- 修复首次并发探活同一模型时可能因缓存唯一键冲突导致任务失败的问题。
- 修复仅支持 Responses 路径的模型在 fallback 时未携带思考模式 `extra_body` 的问题。
- 修复候选探活遇到 HTTP 200 但内容为空时未继续尝试思考模式参数的问题。
- 修复前端依赖锁文件与当前包配置不一致的问题。
- 清理抓取运行时中的一处中文乱码提示和不可达旧代码。
