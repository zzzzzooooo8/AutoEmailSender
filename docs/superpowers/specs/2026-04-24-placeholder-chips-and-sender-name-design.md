# 占位符标签与发件人姓名拆分设计

## 背景

当前邮件模板使用 `{{name}}`、`{{sender_name}}` 等占位符。它们在模板导入、个人页默认模板、批量任务和测试写信中共用同一套渲染逻辑。

现有体验存在 3 个问题：

- 用户在富文本编辑器里直接看到 `{{name}}`，不容易理解这是可替换字段。
- 测试写信页直接发送时只清洗 HTML，不会渲染占位符，可能把 `{{name}}` 原样发出。
- 身份字段 `name` 同时承担「配置名称」和「发件人显示名」两种职责，导致 `From` 和 `{{sender_name}}` 使用的是用户以为只用于管理的配置名。

用户已确认：

- 编辑器里的占位符显示为真正的内联标签。
- 真正拆成「配置名称」和「发件人姓名」两个字段。
- 旧数据迁移时，两个新语义都使用旧的 `name` 值。
- 测试写信中，`{{name}}` 渲染为「测试收件人」。

## 目标

- 在富文本编辑器中用内联标签展示占位符，降低模板语法暴露感。
- 保持底层模板变量格式兼容，仍保存为 `{{name}}`、`{{sender_name}}`。
- 将身份管理名称和邮件发件显示名拆开。
- 测试写信发送前统一渲染 subject、body text、body html 中的占位符。
- 防止测试邮件历史中继续保存未替换的 `{{...}}`。

## 非目标

- 第一版不做真实导师预览切换。
- 第一版不做「标签 -> 当前值」的实时预览模式。
- 第一版不允许用户自定义占位符。
- 第一版不改变正式批量任务的占位符集合。

## 占位符定义

前后端共享同一组占位符定义：

| Key | 编辑器标签 | 含义 |
| --- | --- | --- |
| `name` | 导师姓名 | 正式发送时为导师姓名；测试写信时为「测试收件人」 |
| `email` | 导师邮箱 | 正式发送时为导师邮箱；测试写信时为当前发件邮箱 |
| `title` | 导师职称 | 测试写信时为「测试收件人」 |
| `university` | 导师学校 | 测试写信时为「测试学校」 |
| `school` | 导师学院 | 测试写信时为「测试学院」 |
| `department` | 导师院系 | 测试写信时为「测试院系」 |
| `research_direction` | 研究方向 | 测试写信时为「测试研究方向」 |
| `sender_name` | 发件人姓名 | 来自身份的发件人姓名 |
| `sender_email` | 发件邮箱 | 来自身份的发件邮箱 |

后端仍使用当前 `render_template_string()` 的宽松策略：未知占位符替换为空字符串。前端只提供已知占位符菜单。

## 编辑器设计

### 展示

`EmailTemplateEditor` 新增占位符内联节点：

- 节点显示为圆角标签，例如「导师姓名」。
- 标签整体不可拆分，删除时一次删除整个占位符。
- 标签在编辑器内只显示中文含义，不显示 `{{name}}`。
- 标签颜色与当前红色主色保持一致，但弱化背景，避免抢正文注意力。

### 插入

工具栏新增「占位符」菜单：

- 菜单列出所有可用占位符。
- 用户点击后，在当前光标位置插入对应标签。
- 如果没有选区或光标，插入到编辑器末尾。

### 保存与加载

编辑器对外输出仍保持兼容：

- HTML 中输出 `{{name}}` 形式的文本节点。
- 派生纯文本中也输出 `{{name}}`。
- 加载已有 HTML 或纯文本时，自动把 `{{name}}` 解析为占位符标签。

这样旧模板、导入模板和后端渲染逻辑无需一次性重写。

### 复制粘贴

第一版规则：

- 从编辑器复制到外部时，输出 `{{name}}` 文本。
- 从外部粘贴含 `{{name}}` 的内容时，解析为标签。
- 从外部粘贴未知 `{{custom}}` 时保留普通文本，不显示为标签。

## 身份字段拆分

### 数据模型

在 `identity_profiles` 中新增字段：

- `profile_name`：配置名称，用于身份列表、顶部选择器和管理页显示。
- `sender_name`：发件人姓名，用于邮件 `From` 和 `{{sender_name}}`。

为降低迁移风险，旧字段 `name` 的处理可以按实施阶段决定：

1. 兼容阶段：保留 `name`，将它作为 `profile_name` 的兼容别名输出。
2. 清理阶段：前后端全部迁移后，再考虑删除或停止使用旧字段。

本次第一版只要求完成兼容阶段。

### 迁移规则

旧数据迁移：

- `profile_name = name`
- `sender_name = name`

新建身份时：

- 用户必须填写配置名称。
- 用户必须填写发件人姓名。
- 可在 UI 上提供提示：如果不确定，发件人姓名可以先与配置名称相同。

### 使用规则

- 顶部身份选择器、身份列表、编辑器标题：使用 `profile_name`。
- 邮件 `From`：使用 `sender_name`。
- `{{sender_name}}`：使用 `sender_name`。
- 兼容旧接口时，如果 `sender_name` 为空，则 fallback 到 `name` 或 `profile_name`。

## 测试写信发送渲染

测试写信页发送前，后端必须先渲染占位符，再清洗并发送。

### 渲染上下文

测试上下文固定为：

- `name = 测试收件人`
- `email = identity.email_address`
- `title = 测试收件人`
- `university = 测试学校`
- `school = 测试学院`
- `department = 测试院系`
- `research_direction = 测试研究方向`
- `sender_name = identity.sender_name`
- `sender_email = identity.email_address`

### 发送流程

测试写信发送流程调整为：

1. 接收前端传来的 subject、body text、body html。
2. 使用测试上下文渲染 subject、body text、body html。
3. 对渲染后的 HTML 执行 `normalize_email_html()`。
4. 如果没有 HTML，则把渲染后的纯文本转成 HTML。
5. 发送邮件。
6. 保存发送历史时，保存渲染后的 subject、body text、body html。

这样用户即使直接在测试写信编辑器中看到占位符标签，也不会把原始变量发出去。

## 页面调整

### 个人页身份编辑

发件身份区域拆成两个字段：

- 配置名称：用于系统内选择和管理。
- 发件人姓名：会显示在邮件发件人和 `{{sender_name}}` 中。

模板占位符说明中的 `{{sender_name}}` 文案改为「发件人姓名」。

### 测试写信页

测试写信页仍显示当前身份和收件邮箱，并补充一条轻量提示：

- 测试写信会把导师占位符替换成测试值。
- `{{name}}` 在测试邮件中会替换为「测试收件人」。

提示应保持克制，不占用主编辑区空间。

## 兼容性

### API

第一版保持旧字段兼容：

- 响应中继续提供 `name`，值等同于 `profile_name`。
- 新增 `profile_name` 和 `sender_name`。
- 前端优先使用新字段，旧测试或旧入口可继续读取 `name`。

请求中：

- 新入口提交 `profile_name` 和 `sender_name`。
- 后端短期接受旧的 `name`，并映射到 `profile_name`，避免旧前端请求报错。

### 模板

已有模板无需迁移。编辑器加载后，会将已有 `{{...}}` 显示为标签；保存后仍输出 `{{...}}`。

## 错误处理

- 如果测试发送渲染后 subject 为空，返回「测试邮件需要主题」。
- 如果测试发送渲染后正文为空，返回「测试邮件需要正文」。
- 如果 `sender_name` 为空，后端 fallback 到 `profile_name` 或旧 `name`，避免邮件 From 为空。
- 未知占位符在后端渲染为空字符串；前端不提供未知占位符插入入口。

## 测试策略

### 前端

- `EmailTemplateEditor` 可插入「导师姓名」占位符标签。
- 编辑器加载 `<p>{{name}} 老师您好</p>` 时显示「导师姓名」标签。
- 编辑器输出 HTML 和纯文本时仍包含 `{{name}}`。
- 从菜单插入 `{{sender_name}}` 后，派生 text 包含 `{{sender_name}}`。
- 个人页身份表单展示「配置名称」和「发件人姓名」两个字段。
- 顶部身份选择器使用配置名称。

### 后端

- 身份迁移后旧数据拥有 `profile_name` 和 `sender_name`。
- 创建和更新身份时能保存两个字段。
- `build_template_context()` 使用 `sender_name` 填充 `{{sender_name}}`。
- `build_email_message()` 的 `From` 使用 `sender_name`。
- 测试写信发送会替换 subject、body text、body html 中的 `{{name}}`。
- 测试写信发送历史保存替换后的正文。

### 回归

- 固定模板批量任务仍能渲染导师占位符。
- 模板润色模式仍能把模板正文传给 LLM。
- HTML 模板导入后仍保留 `{{name}}` 等占位符。

## 验收标准

- 用户可以在编辑器中通过菜单插入占位符标签。
- 旧模板中的 `{{name}}` 在编辑器里显示为「导师姓名」标签。
- 保存后的模板仍能被正式发送链路渲染。
- 身份页有独立的配置名称和发件人姓名。
- 测试写信直接发送不会发出原始 `{{name}}`。
- 邮件 `From` 使用发件人姓名。
