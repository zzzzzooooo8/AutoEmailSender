# 导师导入模板说明增强设计

## 背景

导师管理页支持导入 `csv` 和 `xlsx` 文件，但当前模板只有字段名。用户下载后不容易判断每个字段的含义、格式和填写示例，尤其是 `recent_papers` 的分隔规则、必填字段和 URL 字段的用途。

## 目标

- 让用户打开模板文件后即可理解怎么填写，不需要来回查看页面说明。
- CSV 和 XLSX 模板都直接包含字段说明、格式提示和示例行。
- 如果用户没有删除说明行或示例行，导入系统会自动忽略，不产生假导师，也不增加失败条数。
- 保持现有导入字段、按邮箱覆盖更新、回收站自动恢复等规则不变。

## 非目标

- 不改造成多步骤导入向导。
- 不增加逐行错误详情或导入预检功能。
- 不改变导师字段集合和数据库结构。

## 模板结构

### CSV 模板

CSV 模板顶部包含若干说明行，说明行以 `#` 开头。说明行先解释整体导入规则，再列出每个字段的含义和格式。随后是固定字段表头，再放 1 行示例数据。

```csv
# 导师导入模板
# 从字段名下一行开始填写；说明行和示例行可以保留，系统导入时会自动忽略
# 必填字段：name, email
# name：导师姓名，必填。示例：张明远
# email：导师邮箱，必填，必须是邮箱格式。示例：zhang@example.edu
# title：导师职称。示例：教授
# university：学校名称。示例：示例大学
# school：学院名称。示例：人工智能学院
# department：院系或系所。示例：计算机科学系
# research_direction：研究方向，多个方向用中文分号 ； 分隔。示例：大语言模型；智能体；信息抽取
# recent_papers：近期论文，多篇用 | 分隔。示例：Paper A|Paper B
# profile_url：导师主页链接。示例：https://example.edu/zhang
# source_url：数据来源链接。示例：https://example.edu/faculty
name,email,title,university,school,department,research_direction,recent_papers,profile_url,source_url
示例：张明远,zhang@example.edu,教授,示例大学,人工智能学院,计算机科学系,大语言模型；智能体；信息抽取,Paper A|Paper B,https://example.edu/zhang,https://example.edu/faculty
```

### XLSX 模板

XLSX 模板继续使用 `Professors` sheet，顶部先放整体导入规则和字段说明，然后放字段表头和示例行。为了让用户在 Excel 中更容易阅读，说明行使用合并单元格和浅色样式，字段表头加粗。

不额外增加说明 sheet，避免用户在多个 sheet 之间切换。

## 导入解析规则

导入时不再假设第一行就是字段表头，而是自动扫描文件：

1. 跳过空行。
2. 跳过第一列以 `#` 开头的说明行。
3. 找到同时包含全部模板字段的第一行，作为表头行。
4. 从表头行下一行开始读取数据。
5. 自动忽略模板示例行，识别条件为 `name` 以 `示例：` 开头。
6. 空数据行继续忽略，不计入失败数。
7. 真实数据行仍按现有规则校验：`name` 和 `email` 必填，邮箱格式非法则计入失败数。

## 前端导入弹窗

导入弹窗保留现有双栏结构，但说明更轻：

- 左侧说明模板已经内置字段说明和示例行。
- 明确说明示例行可保留，导入时会自动忽略。
- 保留关键规则：`recent_papers` 多篇论文用 `|` 分隔；同邮箱覆盖更新；回收站同邮箱自动恢复。

不在弹窗里放完整字段字典，避免页面重复模板内容。

## 测试

### 后端

- CSV 模板包含 `#` 说明行、字段表头和示例行。
- XLSX 模板包含字段说明、字段表头和示例行。
- 导入带说明行和未删除示例行的 CSV 时，只导入真实数据，示例行不计入失败数。
- 导入带说明行和未删除示例行的 XLSX 时，只导入真实数据。
- 现有字段缺失、邮箱非法、同邮箱覆盖更新规则保持不变。

### 前端

- 打开导入弹窗后，能看到「模板内含说明和示例」「示例行可保留并自动忽略」「recent_papers 使用 `|` 分隔」等关键提示。
- 开始导入前未选择文件时，现有提示仍正常出现。

## 风险与兼容性

- 旧版只有表头的 CSV/XLSX 文件仍可导入，因为扫描逻辑会直接识别第一行表头。
- 用户手动新增的前置说明行不会影响导入，只要后续存在完整字段表头。
- 示例行只通过 `name` 的 `示例：` 前缀识别，避免误跳过测试数据或用户临时使用的 `example.edu` 邮箱。
