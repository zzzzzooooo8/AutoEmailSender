# 批量任务日历式定时发送设计

## 1. 背景

当前批量任务的定时发送只支持“开始时间、结束时间、发送数量”，缺少“哪些日期允许发送”的配置。用户需要能够先通过日期范围快速初选，再在日历上逐日调整，最终按明确的日期集合执行发送。

本设计聚焦创建批量任务时的日期选择、数据模型和后端动态派发规则，不引入 cron、节假日规则或复杂自动排程系统。

## 2. 设计目标

- 支持用户按具体日期选择批量任务发送日。
- 支持用日期范围和快捷规则快速生成初始选择。
- 支持在日历上点击某一天切换选中或未选中。
- 允许用户添加初选范围外的日期。
- 后端执行只依赖最终选中的日期集合，不依赖初选范围或快捷规则。
- 保持现有本地 dispatcher 架构，使用动态派发，不为每封邮件提前固化 `scheduled_at`。

## 3. 非目标

- 不支持 cron 表达式。
- 不支持法定节假日自动避让。
- 不支持为不同日期设置不同发送窗口或不同发送数量。
- 不支持多套发送规则叠加。
- 不重做任务大厅整体信息架构。

## 4. 核心交互

### 4.1 定时发送配置结构

创建批量任务的“定时发送”配置分为三块：

- 快速选择日期
- 日历微调
- 每日发送窗口

“快速选择日期”提供初选工具：

- 开始日期
- 结束日期
- 快捷规则：每天、工作日、周一三五、周末、清空重选

这些输入只用于批量生成日历上的选中日期，不作为后端执行依据。

### 4.2 日历微调

快捷规则生成后，系统在日历上选中匹配日期。用户可以直接点击日期格子切换状态：

- 未选中日期点击后变为选中。
- 已选中日期点击后变回普通未选中状态。
- 被取消的日期不显示划线或特殊历史状态。

日历支持月份切换。若最终选中日期跨多个月，用户通过上个月、下个月切换查看和编辑。

### 4.3 范围外日期

日期范围只是初选工具，不是硬约束。用户可以通过“添加日期”添加范围外日期。添加后：

- 该日期进入最终选中集合。
- 日历可以切换到该日期所在月份。
- 后端按最终日期集合执行。

### 4.4 摘要文案

定时配置下方显示摘要：

`已选 7 天，将在 09:00 至 18:00 之间动态发送，每天最多 20 封。`

任务大厅卡片展示为：

`4/28-5/12 共 8 天，09:00-18:00，每天最多 20 封`

如果日期不连续，仍展示最早日期、最晚日期和总天数，不在卡片中展开全部日期。

## 5. 数据模型

### 5.1 前端类型

`TaskScheduleConfig` 新增：

```ts
scheduledDates?: string[];
```

日期字符串使用 `YYYY-MM-DD` 格式。

### 5.2 API DTO

`CreateBatchTaskRequestDTO` 新增：

```ts
scheduled_dates: string[] | null;
```

`BatchTaskCardDTO` 新增：

```ts
scheduled_dates: string[] | null;
```

### 5.3 后端模型

`batch_tasks` 表新增 JSON 字段：

```py
scheduled_dates
```

存储排序、去重后的 `YYYY-MM-DD` 字符串数组。

现有字段继续保留：

- `schedule_type`
- `window_start_time`
- `window_end_time`
- `emails_per_window`

任务执行语义为：

`scheduled_dates` 中的每个日期，在 `window_start_time` 到 `window_end_time` 之间，每天最多发送 `emails_per_window` 封。

## 6. 校验规则

### 6.1 前端校验

- `schedule_type = immediate` 时，不要求 `scheduledDates`。
- `schedule_type = scheduled` 时，`scheduledDates` 至少包含 1 天。
- `scheduledDates` 提交前去重并升序排序。
- 日期必须是合法 `YYYY-MM-DD`。
- 开始时间和结束时间必须存在。
- 结束时间必须晚于开始时间。
- 每日发送数量必须大于 0。

### 6.2 后端校验

后端不能信任前端，需重复校验：

- `schedule_type` 只能是 `immediate` 或 `scheduled`。
- `scheduled` 类型必须有至少 1 个合法日期。
- 日期数组去重、排序后再入库。
- 时间字段必须是 `HH:mm`。
- `window_end_time` 必须晚于 `window_start_time`。
- `emails_per_window` 必须大于 0。

## 7. 动态派发规则

批量任务使用动态派发，不在创建任务时为每个子任务提前写入 `scheduled_at`。

dispatcher 每次运行时判断：

- 批量任务状态为 `running`。
- 子任务状态允许发送。
- 当前本地日期在 `batch_task.scheduled_dates` 中。
- 当前本地时间位于 `window_start_time` 和 `window_end_time` 之间。
- 当前日期该批量任务已发送数量小于 `emails_per_window`。

满足条件时，选择下一封待发送任务执行。

如果当前日期不在 `scheduled_dates` 中，或当前时间不在窗口内，dispatcher 跳过该批量任务。

## 8. 组件边界

### 8.1 `TaskScheduleSettings`

继续作为发送策略总容器，负责：

- 立即发送和定时发送切换。
- 展示定时发送配置。
- 展示摘要文案。

### 8.2 `TaskDateSelector`

新增组件，负责：

- 日期范围输入。
- 快捷规则按钮。
- 月历展示。
- 日期格子选中状态切换。
- 添加范围外日期。
- 将最终 `YYYY-MM-DD[]` 回传给表单。

### 8.3 `useCreateTaskForm`

新增状态和方法：

- `scheduledDates`
- `setScheduledDates`
- `toggleScheduledDate`
- `applyDateRule`

提交前统一输出排序、去重后的日期数组。

## 9. 错误处理

- 没有选择日期时，创建按钮保持可点但提交后提示：`请至少选择一个发送日期`。
- 初选开始日期晚于结束日期时，提示：`结束日期必须不早于开始日期`。
- 添加日期格式非法时，提示：`请选择有效日期`。
- 如果选中日期已存在，添加操作不重复插入。
- 如果所有日期都被取消，摘要显示 `尚未选择发送日期`。

## 10. 测试策略

### 10.1 前端

- 选择“工作日”时跳过周末。
- 点击已选日期后恢复为普通未选中状态。
- 点击未选日期后加入最终日期集合。
- 可以添加初选范围外日期。
- 提交时日期数组去重、排序。
- `scheduled` 且日期为空时显示校验错误。
- 摘要能正确显示已选天数、日期跨度、发送窗口和每日数量。

### 10.2 后端

- 创建定时批量任务时保存排序后的 `scheduled_dates`。
- `scheduled` 且 `scheduled_dates` 为空时返回 400。
- 非法日期格式返回 400。
- dispatcher 在非选中日期不派发。
- dispatcher 在选中日期但时间窗口外不派发。
- dispatcher 达到当天 `emails_per_window` 后不继续派发。
- dispatcher 在选中日期和窗口内动态派发下一封待发送任务。

## 11. 风险与边界

- “当前日期”和“当前时间”需要使用同一时区口径。当前版本可先使用服务端本地时区，并在后续如有多时区需求时单独设计。
- 任务卡片只显示日期跨度和天数，不能完整表达所有非连续日期；完整日期列表应在任务详情中展示。
- 动态派发依赖当天已发送数量统计，需要明确统计口径是同一批量任务下 `sent_at` 落在当前日期的邮件数量。

## 12. 最终决策

采用“日期范围初选 + 日历点击微调 + 最终日期数组 + 动态派发”的方案。

范围和快捷规则只存在于前端交互中，后端只保存最终 `scheduled_dates`。用户在日历上的选中结果是唯一事实来源。
