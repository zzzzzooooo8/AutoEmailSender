# 匹配度评分量表设计

## 背景

当前匹配度由 LLM 根据默认材料、导师研究方向和近期论文给出 `match_score`。现有提示词已经要求模型只基于可见证据评分，但评分标准不够显式，容易出现两个问题：

- 同类材料在不同导师之间分数波动较大。
- 导师没有近期论文时，模型可能把「论文缺失」误判成「低匹配」。

本设计只调整评分口径，不新增置信度字段，不改变数据库结构。

## 目标

- 让 `match_score` 更稳定、更可解释。
- 允许「无近期论文但研究方向具体」的导师获得高分。
- 对真正信息不足的情况设置分数上限，避免模型在证据薄弱时给出过高分。
- 继续使用现有的 `match_reason`、`fit_points`、`risk_points` 和 `keywords` 承载解释信息。

## 非目标

- 不引入 `evidence_confidence` 等新字段。
- 不保存子项分数。
- 不在本地实现规则打分，仍由 LLM 根据提示词输出最终分数。
- 不改变前端展示结构。

## 评分量表

`match_score` 总分为 100 分，由 3 个维度组成。模型应先按维度判断，再给出总分。

### 研究主题匹配度：0-50 分

衡量学生默认材料与导师研究方向或近期论文是否在研究问题、应用场景或领域上有交集。

- 45-50 分：具体研究问题高度重合。
- 35-44 分：同一方向，有明确交集。
- 20-34 分：宽泛领域相关，但具体问题不同。
- 1-19 分：只有弱相关背景。
- 0 分：看不到相关性。

### 能力与方法匹配度：0-30 分

衡量学生材料中的技能、方法、项目、论文或工具是否能支撑导师方向。

- 25-30 分：能力可以直接支撑导师方向。
- 15-24 分：有部分可迁移能力。
- 5-14 分：只有基础背景或泛化能力。
- 0 分：看不到支撑能力。

### 个性化理由充分度：0-20 分

衡量能否写出具体、可信、不空泛的套磁理由。

- 16-20 分：能提炼出具体匹配点。
- 8-15 分：能写出合理但不够具体的理由。
- 1-7 分：只能泛泛表达兴趣。
- 0 分：无法形成可信理由。

## 上限规则

上限规则用于处理证据不足。模型计算总分后，必须应用以下上限：

- 没有近期论文，但研究方向具体：不限制最高分。
- 没有近期论文，且研究方向很宽泛：`match_score` 最高 75。
- 没有研究方向，但有近期论文：`match_score` 最高 85。
- 研究方向和近期论文都缺失：`match_score` 最高 30。
- 学生默认材料缺少可见研究、项目或技能证据：`match_score` 最高 60。

## 输出解释要求

模型仍输出现有字段：

- `match_reason`：用一句话说明总分的主要依据。
- `fit_points`：列出 1-5 条具体匹配点。
- `risk_points`：列出 0-5 条风险或证据不足点。
- `keywords`：列出 0-6 个关键词。

当触发上限规则时，`risk_points` 必须说明触发原因。例如：

- 「导师没有近期论文，但研究方向较具体，因此未限制最高分。」
- 「导师研究方向较宽泛且缺少近期论文，分数按最高 75 处理。」
- 「默认材料缺少可见项目或技能证据，分数按最高 60 处理。」

## 预期行为

### 无近期论文但方向具体

导师研究方向为「LLM-based biomedical information extraction」，学生默认材料包含信息抽取项目和大模型相关经历。模型可以给出高分，例如 80 分以上，因为研究方向足够具体。

### 无近期论文且方向宽泛

导师研究方向仅为「Artificial Intelligence」或「Data Mining」，学生材料也有 AI 背景。模型可以判断为相关，但最高不超过 75 分，并在 `risk_points` 说明导师信息不足。

### 方向和论文都缺失

导师缺少研究方向和近期论文时，模型最高只能给 30 分。此时 `match_reason` 应强调无法基于可见证据判断。

## 实现边界

后续实现只需要修改匹配分析提示词：

- 在 `SYSTEM_MATCH_ONLY_PROMPT` 中加入评分量表和上限规则。
- 保持 `temperature = 0`。
- 保持返回 JSON 结构不变。
- 保持现有审计记录和 token 统计逻辑不变。

相关代码位置：

`@backend/app/services/llm_runtime.py`

## 测试建议

- 更新 prompt 单测，断言评分量表和上限规则出现在匹配分析系统提示词中。
- 增加无近期论文但研究方向具体的 prompt 构造测试，确保导师近期论文为空时仍保留研究方向。
- 保留现有 LLM runtime 和匹配审计测试，避免破坏 token 统计和落库逻辑。
