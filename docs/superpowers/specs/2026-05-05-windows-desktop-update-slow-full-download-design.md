# Windows 桌面更新慢速切换全量下载与常驻进度设计

## 背景

当前桌面端更新使用 `electron-updater` + GitHub Releases。差分下载在部分网络环境下可能速度很低，且完成后会立刻触发安装，用户会感觉更新过程既慢又打断操作。

## 目标

- 更新进度在界面中常驻可见，不使用模态弹窗阻塞正常操作。
- 增量下载和全量下载都展示实时进度。
- 进度信息至少包含速度、已下载大小、未下载大小和预计剩余时间。
- 下载开始后先尝试差分下载；若持续过慢，则允许切换到全量下载。
- 用户也可以在开始下载前主动选择全量下载。
- 全量下载仍在应用内完成，不打开浏览器。
- 所有下载确认入口都应提示预计消耗的流量。
- 同一版本只下载一次；用户选择稍后安装后，下次进入更新流程应直接复用已下载内容。
- 新版本出现时，旧版本下载缓存应被清理。

## 非目标

- 不改动桌面端以外的业务功能。
- 不引入新的更新分发渠道。
- 不把更新流程改成浏览器手动下载。
- 不实现多更新频道。

## 设计概览

更新流程分为两个阶段：下载阶段和安装阶段。

下载阶段先走差分下载，主进程持续汇报进度。若下载速度长期过低，则只提示一次，允许用户切换为全量下载。切换时取消当前差分下载，再以同一版本的完整安装包重新下载。

用户也可以不等待慢速提示，在开始下载前主动选择全量下载。该入口适合已知当前网络对 GitHub Range 请求较慢、但浏览器或整包下载速度较快的用户。

安装阶段不在下载完成后自动触发，而是进入待安装状态，由用户选择「立即重启安装」或「稍后安装」。

## 状态机

更新状态至少包含：

- `idle`
- `checking`
- `available`
- `downloading`
- `slow_download_offered`
- `downloaded_pending_install`
- `installing`
- `not_available`
- `error`

`downloading` 需要携带：

- 当前版本号
- 目标版本号
- 已下载大小
- 总大小
- 未下载大小
- 实时速度
- 预计剩余时间
- 当前是否为全量下载

`slow_download_offered` 只在同一版本、同一次下载会话里触发一次。

## 流量提示

所有会触发下载的确认入口都应展示预计消耗的流量：

- 发现新版本并询问是否下载时，展示完整安装包大小，并说明差分下载的实际消耗会在下载开始后按进度实时更新。
- 用户主动选择全量下载时，展示完整安装包大小。
- 慢速提示中建议切换全量下载时，展示完整安装包大小和当前差分下载已消耗流量。
- 如果已存在同版本下载缓存，提示不再消耗下载流量，可直接安装。

差分下载的准确流量在开始下载前不一定能可靠获得，因此初始提示以完整安装包大小作为上限说明。下载开始后，常驻状态区使用实际进度展示已下载大小和未下载大小。

## 慢速判断规则

- 从下载开始后第 10 秒开始统计。
- 之后每 30 秒重新计算一次预计剩余时间。
- 预计剩余时间超过 3 分钟时，触发慢速提示。
- 慢速提示只展示一次，不重复打断用户。

预计剩余时间按最近一段时间的平均速度估算。

## 增量切换全量下载

当用户选择全量下载时：

1. 取消当前差分下载。
2. 将当前更新会话标记为全量下载模式。
3. 使用同一版本号重新下载完整安装包。
4. 全量下载继续沿用同一套进度展示逻辑。

这里的全量下载仍然是应用内下载，不打开系统浏览器。

用户可以通过两个入口选择全量下载：

- 开始下载前主动选择全量下载。
- 差分下载过慢时，从慢速提示切换为全量下载。

## 常驻进度展示

更新按钮旁边增加常驻状态区，用于展示：

- 当前版本 / 目标版本
- 预计消耗流量
- 下载进度
- 下载速度
- 已下载大小
- 未下载大小
- 预计剩余时间
- 当前状态：检查中、下载中、待安装、安装中、错误

该区域不得使用模态弹窗遮挡主界面。用户在下载期间仍可继续使用其他页面功能。

## 待安装流程

下载完成后不自动安装，改为进入待安装状态。

用户可选择：

- `立即重启安装`
- `稍后安装`

若选择「稍后安装」，更新文件应保留在本地缓存中。下一次再次点击检查更新时，如果版本未变化，应直接进入安装，不重新下载。

## 缓存与清理

- 同一版本只保留一份更新缓存。
- 新版本出现时，旧版本缓存应被清理。
- 安装完成后，已安装版本对应的缓存应被删除。

缓存位置由主进程管理，放在应用自己的用户数据目录下，避免依赖浏览器下载目录。

## 数据流

1. 渲染进程点击「检查更新」。
2. 主进程调用 `electron-updater` 检查版本。
3. 若有新版本，主进程创建更新会话并开始下载。
4. 主进程持续向渲染进程推送进度事件。
5. 若触发慢速提示，渲染进程只展示一次切换入口。
6. 若用户选择全量下载，主进程取消当前下载并重新开始。
7. 下载完成后，渲染进程展示待安装状态。
8. 用户选择安装后，主进程执行 `quitAndInstall`。

## 错误处理

- 检查更新失败时，保留当前页面可用状态。
- 下载失败时，展示错误摘要，并允许重试。
- 切换全量下载失败时，回退到当前错误态，不清空已有更新状态，方便用户再次重试。
- 安装失败时，保留已下载文件，避免再次下载。

## 测试

需要覆盖以下行为：

- 慢速检测只触发一次。
- 从差分下载切换到全量下载时会取消原下载。
- 下载状态能持续输出速度、已下载大小、未下载大小和预计剩余时间。
- 下载完成后不会立刻安装。
- 选择「稍后安装」后，再次检查同版本更新会复用已下载文件。
- 新版本出现时旧缓存会被清理。

## 验收标准

- 用户能在更新按钮旁持续看到下载进度。
- 下载过程不会阻塞应用主界面。
- 增量下载过慢时，用户可切换到全量下载。
- 全量下载和增量下载都能复用同一套进度展示。
- 同一版本不会被重复下载。
- 新版本到来时，旧版本缓存会被删除。
