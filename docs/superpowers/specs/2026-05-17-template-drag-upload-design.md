# 默认模板拖拽上传设计

## 目标

在“默认发信模式与默认模板”弹窗中支持拖拽上传套磁信模板文件。用户拖入文件后，系统复用现有模板导入接口解析文件，并自动把解析出的正文回填到“默认模板正文”编辑器。

## 范围

- 入口限定在 `ProfilePage` 的 `OutreachTemplateModal`。
- 文件格式沿用现有支持范围：`.docx`、`.html`、`.htm`、`.txt`、`.md`。
- 后端解析能力、数据结构和保存逻辑保持不变。
- 本次只导入正文，不从文件中推断主题；主题仍由用户单独填写。

## 交互

弹窗顶部的“导入默认模板文件”区域和“默认模板正文”编辑器都支持模板文件拖拽：

- 点击区域时打开文件选择器。
- 拖拽文件进入顶部导入区域时显示高亮状态。
- 拖拽文件到正文编辑器时直接识别为模板导入，不触发编辑器或浏览器默认的文件路径插入行为。
- 拖拽离开或完成上传后恢复普通状态。
- 正在导入时禁用重复触发，并显示加载状态。

如果当前默认模板正文已有内容，拖拽或点击导入前先弹出确认：

- 标题：确认覆盖默认模板正文？
- 描述：导入模板文件会替换当前正文内容，主题不会被修改。
- 确认后继续导入并覆盖正文。
- 取消则不调用导入接口。

## 数据流

1. 用户点击顶部导入区域，或拖拽文件到顶部导入区域/默认模板正文编辑器。
2. 前端确认是否需要覆盖已有正文。
3. 调用现有 `importIdentityTemplate(file)`。
4. 成功后把 `body_text` 写入 `outreach_template_body_text`，把 `body_html` 写入 `outreach_template_body_html`。
5. `EmailTemplateEditor` 通过已有 `html` prop 自动刷新编辑器内容。

## 错误处理

- 无文件时不触发导入。
- 正在导入时忽略新的拖拽或选择。
- 不支持的格式和解析失败沿用现有后端错误提示。
- 切换身份后返回的旧导入结果沿用现有 `identityEditorIdRef` 保护，不覆盖新身份。

## 测试

- `ProfilePage.test.ts` 断言模板弹窗包含拖拽处理器和拖拽状态。
- `ProfilePage.test.ts` 断言导入前会在正文已有内容时调用确认弹窗。
- 运行 `npm.cmd run test:node -- ProfilePage.test.ts` 验证红绿过程。
