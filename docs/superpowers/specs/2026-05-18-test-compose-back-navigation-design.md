# 测试写信页返回上一级设计

## 背景

测试写信页是从个人中心配置流程进入的独立页面，用于给当前身份自己的邮箱发送测试邮件。当前页面顶部只有“测试邮件工作台”“测试写信”和页面说明，用户进入后如果想回到上一页，需要依赖顶部导航或浏览器/桌面窗口的历史返回能力，页面自身缺少明确的返回入口。

这会让测试写信流程出现一个小断点：用户完成测试、或临时想回到个人中心继续调整身份、模板、模型设置时，页面没有一个符合直觉的“返回”动作。

## 已确认的产品决策

- 在测试写信页增加一个轻量返回入口。
- 返回入口文案只显示“返回”。
- 点击后返回浏览历史中的上一级页面。
- 不固定跳转到首页。
- 不把该入口做成主要操作按钮，避免和“生成测试草稿”“保存草稿”“发送测试邮件”竞争视觉层级。

## 目标

- 让用户在测试写信页内能直接返回进入该页面前的上一页。
- 让返回入口的位置和现有页面层级一致，靠近页面标题区域。
- 保持现有测试写信页的业务逻辑、数据加载、草稿保存、发送动作不变。
- 保持现有顶部全局导航、底部发送操作栏不变。

## 非目标

- 不新增面包屑导航。
- 不新增“返回首页”或“返回个人中心”的固定跳转。
- 不调整测试写信页的表单结构、侧栏信息卡、附件选择或发送历史。
- 不修改后端 API、数据库结构或测试写信数据模型。
- 不改变个人中心进入测试写信页的入口文案和逻辑。

## 交互设计

返回入口放在测试写信页头部左侧，位于“测试邮件工作台”标签上方或同一头部区域的第一行，视觉接近现有导师详情页中的返回入口：

```text
← 返回

测试邮件工作台
测试写信
发送到自己的邮箱，用于检查模板、附件和发信设置。
```

点击“返回”后执行路由历史后退：

```text
navigate(-1)
```

返回目标由用户进入测试写信页前的实际历史记录决定。例如：

- 从个人中心进入测试写信页，则返回个人中心。
- 从其他内部页面进入测试写信页，则返回对应的上一页。
- 直接打开测试写信页时，不额外强制跳到首页；直接打开场景没有明确上一级，不在本次需求中扩展 fallback 规则。

## 视觉设计

- 使用左箭头图标加“返回”文字。
- 字号、颜色和间距保持轻量，默认使用中性灰。
- hover 或 focus 时使用项目主色强调。
- 点击区域需要足够清晰，但不使用红色填充样式。
- 返回入口不应挤压右侧身份、模型、测试收件邮箱信息卡。

建议样式方向：

- `inline-flex items-center gap-2`
- `text-sm font-medium text-stone-500`
- `hover:text-primary`
- `focus-visible` 提供清晰轮廓

## 前端实现范围

主要修改文件：

- `frontend/src/pages/TestComposePage.tsx`

建议实现方式：

1. 从 `react-router-dom` 引入 `useNavigate`。
2. 从 `lucide-react` 复用 `ArrowLeft` 或同类左箭头图标。
3. 在 `TestComposePage` 内部初始化 `const navigate = useNavigate()`。
4. 在加载成功后的页面 header 中加入一个 `button type="button"`。
5. 点击按钮时调用 `navigate(-1)`。
6. 保持现有加载态、缺少身份/模型态、发送操作栏逻辑不变。

## 可访问性

- 返回入口使用原生 `button`，不要使用不可聚焦的 `div`。
- 按钮可通过键盘 Tab 聚焦并用 Enter/Space 触发。
- 文案“返回”本身已经表达动作，不需要额外可见说明。
- 保留明确的 `focus-visible` 状态，方便键盘用户识别当前焦点。

## 测试与验收

本次改动属于窄范围前端交互调整。建议验证：

- 从个人中心点击“进入测试写信页”后，测试写信页顶部出现“返回”。
- 点击“返回”后回到进入测试写信页前的页面。
- 返回入口不显示为“返回首页”。
- 测试写信页原有加载、生成草稿、保存草稿、发送测试邮件、附件选择和发送历史展示不受影响。
- 桌面端窗口中按钮样式不与顶部导航或底部发送栏重叠。

## 规格自检

- 行为符合“返回上一级”，没有固定跳转首页。
- 文案为“返回”，没有出现“返回首页”。
- 改动范围限制在测试写信页前端 UI。
- 不涉及后端、数据库、Electron 主进程或发送链路。
