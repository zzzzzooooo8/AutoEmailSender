# 导出导师信息设计规格

## 背景

导师档案管理页目前支持 3 种导师录入方式：智能抓取、模板导入和单个新增。模板导入入口会打开「导入导师文件」弹窗，用户可以在弹窗内下载 XLSX 或 CSV 模板，再按模板字段批量导入导师信息。

现在需要新增「导出导师信息」能力。导出的文件应支持 XLSX 和 CSV，字段格式与现有导入模板保持一致，便于用户备份、外部整理或修改后再次导入。

## 目标

- 在导师档案管理页提供清晰可见的导出入口。
- 导出全部正常导师，不包含回收站导师。
- 导出结果不受当前搜索、筛选、分页或勾选状态影响。
- 导出文件支持 XLSX 和 CSV 两种格式。
- 导出字段顺序和含义与现有导入模板一致。
- 导出的文件在用户不做任何修改的情况下，可以直接重新导入系统，且不产生解析错误或数据格式问题。
- 交互风格与现有「导入导师文件」弹窗保持一致。

## 非目标

- 不支持导出当前筛选结果。
- 不支持导出用户勾选的部分导师。
- 不支持导出回收站导师。
- 不新增字段映射配置、自定义列选择或高级导出选项。
- 不改变现有导入模板下载位置；模板下载仍保留在「导入导师文件」弹窗中。

## 页面入口设计

### 布局位置

导出入口放在现有「导师录入方式」面板右侧。

面板内部调整为左右布局：

- 左侧保留现有 3 个录入入口：智能抓取、模板导入、单个新增。
- 右侧新增独立卡片：导出导师信息。

左侧 3 个录入卡片可以略微压缩宽度，以给右侧导出卡片留出稳定空间。导出卡片与录入卡片处于同一大面板中，但视觉上应保持独立，避免让用户误解导出是第 4 种「录入方式」。

### 卡片内容

右侧卡片建议使用以下信息层级：

- 标题：导出导师信息
- 说明：导出全部正常导师，字段与导入模板一致。
- 主按钮：导出导师信息

视觉上可使用偏绿色的备份 / 导出语义，与「模板批量新增」当前偏黄色的导入语义形成区分。

### 响应式行为

在桌面宽屏下使用左右布局。窄屏或空间不足时，导出卡片应自然换行到录入卡片下方，保持按钮可点击、说明可读，不要求固定在右侧。

## 弹窗设计

点击「导出导师信息」后打开导出弹窗。弹窗结构参考现有「导入导师文件」弹窗，但内容聚焦导出说明和格式选择。

### 弹窗标题和说明

- 标题：导出导师信息
- 描述：将全部正常导师导出为表格文件。字段顺序与导入模板保持一致，便于备份、外部整理或修改后再次导入。

### 格式选择

弹窗内提供 2 个主要操作：

- 导出 XLSX
- 导出 CSV

XLSX 可以作为主按钮样式，CSV 使用次级按钮样式，与现有「导入导师文件」弹窗中的模板下载按钮保持一致。

### 说明信息

弹窗中应明确展示以下说明：

- 导出范围：全部正常导师，不包含回收站导师。
- 当前搜索、筛选、分页和勾选状态不会影响导出结果。
- 字段顺序与导入模板一致，空值保留为空单元格。

## 数据范围和格式

### 导出范围

导出范围固定为全部正常导师，即管理列表中 `archived = false` 的导师记录。

导出不读取前端当前筛选状态，不使用当前分页范围，也不使用用户勾选的导师 ID。

### 字段格式

导出字段应复用现有导入模板的字段定义和顺序，避免形成两套表格协议。字段说明、列名、空值处理应与导入逻辑兼容。

如果导入模板中包含说明行或示例行，导出的真实数据文件不应包含说明行和示例行，只导出表头和导师数据。

导出的文件必须满足「原样再导入」约束：用户下载后不做任何修改，直接通过现有「导入导师文件」流程重新导入时，应能被系统正常解析。由于导入逻辑按邮箱覆盖记录，原样再导入的预期结果是更新现有导师记录，不新增重复导师，不产生字段解析错误。

为满足该约束，导出时应特别注意：

- 表头必须使用导入解析器认可的字段名。
- 必填字段 `name` 和 `email` 必须保持为可导入值。
- `email` 不应被表格格式化破坏。
- `recent_papers` 必须按导入解析器支持的 `|` 分隔格式输出。
- 空值应输出为空字符串，而不是 `None`、`null` 或其他占位文本。
- CSV 必须使用 UTF-8 with BOM，保证中文内容在常见表格软件中打开后仍可保存并导入。

### 文件格式

- XLSX：适合作为默认推荐格式，便于普通用户直接用表格软件打开。
- CSV：适合脚本处理、跨工具导入和轻量交换。

CSV 文件应使用 UTF-8 编码，避免中文字段乱码。

## API 设计

新增导师导出下载接口，建议路径为：

```text
GET /api/professors/export?format=xlsx
GET /api/professors/export?format=csv
```

接口行为：

- `format` 仅接受 `xlsx` 或 `csv`。
- 仅导出全部正常导师。
- 返回附件下载响应。
- 文件名应包含业务语义和日期，例如 `professors-20260520.xlsx`。

前端 API helper 可提供：

```typescript
getProfessorExportDownloadUrl(format: "xlsx" | "csv")
```

前端沿用现有模板下载的触发方式，通过创建临时 `a` 标签触发浏览器下载。

### 下载交互约束

之前模板下载曾出现过「点击下载时打开空白页面」的问题。本功能必须避免该回归。

前端触发下载时应复用当前 `triggerDownload(url)` 行为：

- 创建临时 `a` 标签。
- 设置 `href` 为后端下载 URL。
- 不设置 `target="_blank"`。
- 不设置 `rel="noreferrer"`。
- 点击后立即移除临时标签。

导出 XLSX 和 CSV 的前端测试应覆盖这一点，确保点击导出按钮不会打开新窗口或空白页。

## 后端实现方案

后端可复用现有导师导入模板能力，避免维护两套字段协议。

### 代码组织

建议在 `app.services.professor_management` 中新增导出构建函数，例如：

```python
def build_professor_export(professors: Sequence[Professor], format_name: str) -> tuple[bytes, str, str]:
    ...
```

该函数应复用 `PROFESSOR_TEMPLATE_COLUMNS` 作为表头顺序，并复用当前 XLSX / CSV 生成依赖。

### 数据查询

导出路由只查询正常导师：

```python
select(Professor)
  .where(Professor.archived_at.is_(None))
  .order_by(Professor.updated_at.desc(), Professor.created_at.desc())
```

排序建议与管理列表保持一致，方便用户对照页面数据。

### 行数据转换

每位导师导出为 1 行，列顺序与 `PROFESSOR_TEMPLATE_COLUMNS` 一致：

- `name`
- `email`
- `title`
- `university`
- `school`
- `department`
- `research_direction`
- `recent_papers`
- `profile_url`
- `source_url`

`recent_papers` 应按现有导入格式用 `|` 拼接，确保导出的文件可以作为后续导入基础。

### 响应格式

接口返回 `Response` 附件下载，与现有 `/api/professors/template` 保持一致：

- CSV media type：`text/csv; charset=utf-8`
- CSV 内容：UTF-8 with BOM，降低 Excel 打开中文乱码概率。
- XLSX media type：`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
- `Content-Disposition`：`attachment; filename="professors_export_YYYYMMDD.<ext>"`

如果没有正常导师，后端仍返回只有表头的文件，方便用户确认导出功能可用。

## 状态和错误处理

### 前端状态

需要新增导出弹窗打开状态，例如 `exportModalOpen`。

由于下载通过浏览器文件下载触发，前端可以不维护长时间 loading 状态。若后续实现改为 `fetch` blob 下载，再补充导出中状态和错误提示。

### 后端错误

后端应处理以下情况：

- 不支持的 `format` 返回 400。
- 导出生成失败返回 500，并记录可诊断日志。
- 没有正常导师时，仍返回只有表头的空文件，而不是报错。

## 测试计划

### 前端测试

- 验证导师管理页展示「导出导师信息」入口。
- 验证点击入口后打开导出弹窗。
- 验证弹窗包含「导出 XLSX」「导出 CSV」按钮和范围说明。
- 验证点击格式按钮会调用对应下载 URL。
- 验证点击导出按钮不会设置 `target="_blank"`，避免下载时打开空白页面。

### 后端测试

- 验证 `format=xlsx` 返回 XLSX 附件。
- 验证 `format=csv` 返回 UTF-8 CSV 附件。
- 验证导出只包含正常导师，不包含回收站导师。
- 验证不支持的格式返回 400。
- 验证空数据时返回仅含表头的文件。
- 验证 CSV 使用 UTF-8 BOM，并且 `recent_papers` 按 `|` 拼接。
- 验证导出的 XLSX 和 CSV 在不修改内容的情况下，可以被 `parse_professor_import_file` 正常解析。
- 验证原样再导入不会新增重复导师，而是按邮箱覆盖现有导师记录。

## 验收标准

- 用户能在导师档案管理页现有录入面板右侧找到「导出导师信息」。
- 点击入口后弹出导出弹窗，而不是直接下载。
- 弹窗可以选择 XLSX 或 CSV。
- 导出文件包含全部正常导师，不包含回收站导师。
- 导出不受当前搜索、筛选、分页或勾选状态影响。
- 导出字段与现有导入模板一致，可作为备份和再次导入的数据基础。
- 用户不修改导出文件并重新导入时，系统可以正常解析并按邮箱覆盖现有记录，不出现报错或重复新增。
