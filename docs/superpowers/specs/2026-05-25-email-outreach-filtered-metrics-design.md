# 邮件触达筛选统计设计

## 背景

统计面板中的「邮件触达」需要从固定总览，调整为可按学校、学院和时间范围查看的触达统计。用户希望保留现有的已发送、已回复卡片视觉，不新增回复趋势图，只让筛选条下方的邮件触达组件一起响应筛选。

本设计参考 sub2api 仪表盘的筛选思路：筛选条件作为统计模块的上下文，筛选后卡片和趋势图一起更新；默认无筛选时展示全部数据。

## 目标

1. 在「邮件触达」标题下方新增筛选条。
2. 筛选条下方的邮件触达组件全部受筛选影响：
   - 已发送邮件
   - 已回复
   - 回复率
   - 发送趋势
3. 发送趋势只展示发送和回复，不展示失败。
4. 回复率按导师去重计算，不按邮件数或回复次数重复计算。
5. 默认不筛选时展示全部学校、全部学院、全部时间的数据。

## 非目标

1. 不修改「已发送邮件」和「已回复」卡片的视觉结构。
2. 不新增单独的回复趋势图。
3. 不做学校/学院回复率排行榜、热力图、气泡图或象限图。
4. 不改变导师概览模块现有筛选交互，除非复用组件时需要抽出公共筛选控件。
5. 不把发送失败计入发送趋势展示。

## 页面结构

「邮件触达」模块调整为：

```text
邮件触达
发送进度、回复效果和触达趋势

[时间范围] [学校] [学院] [清空筛选]

[已发送邮件] [已回复] [回复率]

发送趋势
发送 / 回复
```

筛选条放在模块标题下方、统计卡片上方。筛选条以下的所有邮件触达内容使用同一组筛选条件。

## 筛选规则

### 学校筛选

- 默认值为全部学校。
- 选中学校后，学院筛选只展示该学校下可选学院。
- 清空学校时，同时清空学院。

### 学院筛选

- 默认值为全部学院。
- 未选择学校时，学院选择器置灰或展示「请先选择学校」。
- 学院值使用导师资料中的 `Professor.school`。

### 时间筛选

- 默认值为全部时间。
- 支持预设时间段和自定义日期范围。
- 建议预设：
  - 最近 7 天
  - 最近 30 天
  - 最近 90 天
  - 本月
  - 上月
  - 全部时间
  - 自定义范围
- 时间筛选按发送时间归属触达批次：选择某个时间范围时，统计该时间范围内成功发送过的导师，以及这些导师中的已回复导师。
- 如果一个导师在筛选时间范围内有至少一次成功发送，则进入分母。
- 如果该导师已经有过至少一次回复，则进入分子；回复发生时间不要求落在筛选时间范围内。

选择按发送时间归属，是因为这里要回答的是「这段时间触达的人最终有多少回复」，而不是「这段时间收到多少封回复」。

## 统计口径

### 已发送邮件

- 展示筛选范围内的成功发送邮件数量。
- 这里仍按邮件/发送动作计数，不按导师去重。
- 数据来源优先使用 `EmailLog.direction = sent`。
- 对没有 sent 日志但任务已处于 sent/reply_detected 或存在 `sent_at` 的历史数据，保留现有 fallback 兼容。

### 已回复

- 展示筛选范围内已回复导师数量。
- 按导师去重。
- 同一导师回复多封，只计 1。
- 同一导师多次发送后回复，也只计 1。
- 该导师必须在当前筛选范围内至少成功发送过一次，才纳入当前筛选下的已回复统计。

### 回复率

回复率公式：

```text
回复率 = 已回复导师数 / 已触达导师数
```

- 已触达导师数：筛选范围内至少成功发送过一次邮件的导师数，按导师去重。
- 已回复导师数：已触达导师中至少有一次回复的导师数，按导师去重。
- 分母为 0 时，回复率显示 0% 或空状态提示，具体沿用现有卡片风格。

### 发送趋势

- 只展示发送和回复两组数据。
- 不展示失败。
- 发送值：每个时间桶内成功发送邮件数量。
- 回复值：每个时间桶内发生回复的导师数量，按导师在该时间桶内去重。
- 趋势图受学校、学院和时间范围筛选影响。
- 空数据时展示轻量空状态。

## 趋势时间粒度

趋势图根据时间范围自动选择粒度：

- 31 天以内：按天。
- 32 天到 180 天：按周。
- 超过 180 天或全部时间跨度较长：按月。

如果是「全部时间」，后端可根据当前数据最早发送时间到当前日期自动决定粒度。

前端展示时：

- 按天：横轴显示月/日。
- 按周：横轴显示周起始日期。
- 按月：横轴显示年/月。

## 后端接口设计

当前接口：

```text
GET /api/dashboard/overview
```

需要增加参数：

```text
start_date?: YYYY-MM-DD
end_date?: YYYY-MM-DD
```

已有导师概览参数继续使用：

```text
identity_id
llm_profile_id
university?
school?
```

邮件触达筛选使用独立参数，避免筛选邮件触达时连带改变导师概览：

```text
email_university?
email_school?
```

响应中的 `email.summary` 建议扩展或明确字段口径：

```text
sent_count              成功发送邮件数
replied_count           已回复导师数
reply_rate              已回复导师数 / 已触达导师数
contacted_professor_count  已触达导师数
```

`contacted_professor_count` 可用于回复率 tooltip 或辅助文案，例如「12 / 40 位导师已回复」。

趋势字段建议从 `trend_30_days` 改为更通用名称，例如：

```text
trend
```

每个桶包含：

```text
date: string
label: string
sent_count: number
replied_count: number
```

为了减少破坏性，也可以短期保留 `trend_30_days` 字段名，但语义会变成筛选范围趋势。实现时优先以测试和改动范围决定。

## 前端交互设计

### 筛选条

筛选条使用当前项目的白底、石色边框、圆角输入风格，和导师概览筛选保持一致。

控件顺序：

1. 时间范围
2. 学校
3. 学院
4. 清空筛选

时间范围变化、学校变化、学院变化后自动重新请求统计。不在邮件触达筛选条内提供刷新统计按钮。

### 卡片

已发送邮件、已回复卡片视觉不变。

回复率卡片可保留现有视觉，但 helper 文案建议改为：

```text
{已回复导师数} / {已触达导师数} 位导师
```

这样能明确回复率不是按邮件数计算。

### 发送趋势

标题从「近 30 天发送 / 回复 / 失败趋势」调整为：

```text
发送趋势
```

图例只保留：

```text
发送
回复
```

图表仍可先使用当前手绘双柱样式，后续如引入图表库再升级 tooltip 和 hover。若当前实现成本可控，建议补充 hover tooltip，显示当前桶的发送数、回复导师数和日期范围。

## 数据处理边界

1. 学校或学院为空时，后端使用「学校未填写」「学院未填写」作为筛选和展示标签，保持与导师概览一致。
2. 同一导师多封回复只算一个已回复导师。
3. 同一导师在筛选时间范围内多次发送，已触达导师数只算一个。
4. 已发送邮件卡片仍按邮件数显示，因此可能大于已触达导师数。
5. 若导师在时间范围外发送、时间范围内回复，不计入该时间范围的回复率分子；因为其触达批次不属于当前时间范围。
6. 若导师在时间范围内发送、时间范围外回复，计入该时间范围的回复率分子；因为要衡量该时间范围触达批次的最终回复效果。

## 测试计划

### 后端

- 新增或更新 dashboard 统计测试：
  - 同一导师多次回复只计 1 个已回复导师。
  - 同一导师多次发送只计 1 个已触达导师。
  - 已发送邮件数仍按发送日志计数。
  - 学校筛选会限制邮件统计。
  - 学院筛选会限制邮件统计。
  - 时间筛选按发送时间归属。
  - 发送趋势不返回或不使用失败展示数据。

### 前端

- 更新 `DashboardPage` 测试：
  - 邮件触达筛选条存在于标题下方。
  - 修改学校/学院/时间会带参数请求 dashboard overview。
  - 已发送、已回复、回复率卡片展示筛选后的数据。
  - 发送趋势图例不再包含失败。
  - 默认无筛选时请求不携带学校、学院和时间参数。

## 验收标准

1. 打开统计面板时，邮件触达默认展示全部数据。
2. 在邮件触达筛选条选择学校后，已发送邮件、已回复、回复率和发送趋势同步变化。
3. 选择学院后，统计进一步收窄到该学院。
4. 选择时间范围后，统计只基于该时间范围内触达的导师和发送邮件。
5. 同一导师多封回复不会导致回复率超过真实导师回复比例。
6. 发送趋势不再出现失败图例和失败柱。
7. 已发送邮件和已回复卡片视觉结构保持不变。
