# 导师概览重构设计

## 背景

当前统计面板已经有“导师概览”和“邮件触达”两个模块。导师概览第一版以多个指标卡、图表卡和行动列表卡堆叠呈现，信息完整但布局偏散。后续已经移除“高分未联系”“高分未联系导师”“待补全导师”等待行动模块，导师概览需要重构为更像仪表盘的分析区。

参考 `Wei-Shaw/sub2api` 的统计面板，新的导师概览需要强化两个体验：

- 用筛选条件驱动统计图表变化，而不是只展示静态总览。
- 饼图在鼠标悬浮时提供清晰的名称、数量和占比反馈。

## 目标

- 将导师概览改成三层布局：核心指标和学校分布、筛选条、分析图表。
- 顶部保留 3 个同尺寸指标卡：导师总数、已匹配导师、高匹配导师。
- 顶部右侧新增更大的“学校分布”饼图卡，展示全局导师学校结构。
- 中间新增学校 / 学院筛选条。
- 筛选条影响“匹配分数分布”和“资料完整度概览”两个分析图表。
- “匹配分数分布”继续使用当前柱状图。
- “资料完整度概览”从进度条列表改成饼图。
- 饼图 hover 交互仿照 `sub2api`：悬浮扇区显示标签、数量、占比，并高亮当前项。

## 非目标

- 不重做邮件触达模块。
- 不恢复已经删除的高分未联系、待补全导师、高价值待跟进邮件等行动列表。
- 不引入复杂报表能力，例如任意字段分组、导出图表、保存筛选方案。
- 不让学校分布饼图跟随学校 / 学院筛选变化。学校分布保持全局，用来呈现导师库结构。
- 不要求第一版引入大型图表库。可以使用 SVG / React 自绘饼图实现需要的 hover 交互。

## 目标布局

导师概览整体采用一个 4 列网格。右侧学校分布卡跨两行，左侧三张指标卡位于第一行，筛选条位于第二行。

大屏布局：

```text
[导师总数] [已匹配导师] [高匹配导师] [学校分布饼图]
[学校 / 学院筛选条                      ] [学校分布饼图]
[匹配分数分布柱状图      ] [资料完整度饼图          ]
```

中屏布局：

```text
[导师总数] [已匹配导师]
[高匹配导师] [学校分布饼图]
[学校 / 学院筛选条]
[匹配分数分布柱状图]
[资料完整度饼图]
```

小屏布局：

```text
[导师总数]
[已匹配导师]
[高匹配导师]
[学校分布饼图]
[学校 / 学院筛选条]
[匹配分数分布柱状图]
[资料完整度饼图]
```

## 组件结构

### 指标卡

保留当前 `MetricCard` 的信息结构，但导师概览只渲染三张：

- 导师总数：全局未归档导师数。
- 已匹配导师：当前身份和模型下有匹配分的导师数。
- 高匹配导师：匹配分不低于当前身份阈值的导师数。

指标卡不跟随学校 / 学院筛选变化。它们代表当前导师库和当前身份模型的全局概览。

### 学校分布饼图卡

学校分布卡放在导师概览右侧，视觉权重高于普通指标卡。

数据口径：

- 使用全部未归档导师。
- 按 `Professor.university` 聚合。
- 空学校名称统一归类为“学校未填写”。
- 饼图和图例展示所有学校，不合并为“其他”。学校较多时允许扇区变多，图例列表使用滚动区域保持卡片尺寸稳定。

展示结构：

- 卡片标题：学校分布。
- 左侧或上方为饼图 / 环形图。
- 右侧或下方为图例列表：学校名、导师数量、占比。
- 图例列表需要处理长学校名，使用截断加 `title`。

hover 交互：

- 鼠标悬浮扇区时，当前扇区加深或轻微外扩。
- 其他扇区降低透明度。
- 显示浮层：学校名、导师数、占比。
- 悬浮图例行时，也联动高亮对应扇区。

### 学校 / 学院筛选条

筛选条是横向长条卡片，放在三张指标卡下方。

控件：

- 学校选择：默认“全部学校”。
- 学院选择：默认“全部学院”。
- 清空按钮：当任一筛选不为全部时显示。

联动规则：

- 选择学校后，学院选项只显示该学校下的学院。
- 清空学校时，学院也回到“全部学院”。
- 只选择学院但不选择学校时，允许按学院名称跨学校筛选；如果实现复杂，第一版可以要求先选学校再选学院。

推荐第一版采用更稳定的规则：

```text
学校为一级筛选；学院依赖学校。
未选择学校时，学院控件禁用或显示“请先选择学校”。
```

筛选影响范围：

- 影响匹配分数分布。
- 影响资料完整度概览。
- 不影响三张指标卡。
- 不影响学校分布饼图。
- 不影响邮件触达模块。

### 匹配分数分布

保留当前柱状图样式和分桶：

- 未分析
- 0-59
- 60-69
- 70-79
- 80-89
- 90-100

筛选后口径：

- 只统计当前筛选范围内的导师。
- 有最新匹配分的导师进入对应分数桶。
- 没有匹配任务或 `match_score is null` 的导师进入“未分析”。

卡片右上角显示当前筛选摘要：

```text
全部导师
示例大学
示例大学 / 计算机学院
```

### 资料完整度概览饼图

当前资料完整度是多项进度条。重构后改成饼图，但需要避免一个导师同时属于多个扇区导致占比难以理解。

推荐第一版使用互斥分类：

- 完整资料
- 缺邮箱
- 缺研究方向
- 缺近期论文
- 缺主页链接
- 多项缺失

分类优先级：

1. 如果满足完整资料定义，归为“完整资料”。
2. 如果缺失字段数量大于 1，归为“多项缺失”。
3. 如果只缺一个字段，归入对应缺失项。

完整资料定义沿用现有口径：

```text
有邮箱 + 有研究方向 +（有近期论文 或 有主页链接）
```

hover 交互：

- 与学校分布饼图一致。
- 悬浮显示分类名、导师数、占比。
- 图例行和扇区双向高亮。

## 后端接口设计

当前接口：

```text
GET /api/dashboard/overview?identity_id=1&llm_profile_id=2
```

建议扩展为：

```text
GET /api/dashboard/overview?identity_id=1&llm_profile_id=2&university=示例大学&school=计算机学院
```

新增可选参数：

- `university`: 学校名称。
- `school`: 学院名称。

后端构建时保留两套导师集合：

- `all_professors`: 所有未归档导师，用于三张指标卡和学校分布。
- `filtered_professors`: 经过学校 / 学院筛选后的导师，用于匹配分数分布和资料完整度概览。

这样可以保证“学校分布是全局结构图，筛选条是分析视角”的语义稳定。

## 数据模型调整

需要调整导师统计返回结构，建议在 `DashboardMentorSectionRead` 中新增或修改字段：

```text
school_distribution: list[DashboardSchoolDistributionRead]
school_filters: list[DashboardSchoolFilterRead]
active_filter: DashboardMentorFilterRead
filtered_match_score_distribution: list[DashboardMentorMatchBucketRead]
filtered_profile_completeness_distribution: list[DashboardProfileCompletenessBucketRead]
```

为了降低改动量，也可以复用当前字段：

- `match_score_distribution` 直接返回筛选后的分布。
- `profile_completeness` 改为筛选后的饼图分类数据。
- `school_distribution` 保持全局。

推荐第一版使用复用字段，减少前后端 DTO 变动，但将 `profile_completeness` 的含义从“多项进度条”改成“互斥完整度分类”需要同步更新字段命名，避免语义混乱。

更清晰的方案是新增：

```text
profile_completeness_distribution
```

并保留旧 `profile_completeness` 一段时间，前端新页面只使用新字段。

## 前端状态流

前端新增状态：

```text
selectedUniversity: string | null
selectedSchool: string | null
hoveredSchoolDistributionKey: string | null
hoveredProfileCompletenessKey: string | null
```

当用户修改学校 / 学院筛选：

1. 更新本地筛选状态。
2. 重新调用 `getDashboardOverview`，携带 `university` 和 `school`。
3. 页面进入局部加载状态。
4. 替换匹配分数分布和资料完整度饼图数据。

第一版可以刷新整个 overview。若后续性能或视觉闪动明显，再拆分导师分析接口。

## 空状态

- 学校分布没有数据：显示“暂无学校分布数据”。
- 筛选后没有导师：匹配分数分布和资料完整度概览都显示“当前筛选下暂无导师”。
- 学校有数据但学院为空：学院控件禁用，显示“暂无学院”。

## 测试计划

后端：

- 未带筛选参数时，返回全局导师分析数据。
- 带 `university` 时，匹配分数分布只统计该学校导师。
- 带 `university` + `school` 时，匹配分数分布只统计该学校该学院导师。
- 学校分布始终使用全局导师，不受筛选影响。
- 资料完整度互斥分类总数等于筛选后的导师总数。

前端：

- 渲染三张指标卡和学校分布饼图。
- 渲染学校 / 学院筛选条。
- 选择学校后调用 dashboard API，并带上学校参数。
- 选择学院后调用 dashboard API，并带上学校和学院参数。
- 学校分布卡在筛选后仍保留全局数据。
- 饼图 hover 显示名称、数量、占比。

## 审查问题

- 学院筛选是否必须允许“不选学校直接按学院跨学校筛选”？推荐第一版不允许。
- 资料完整度饼图是否采用互斥分类？推荐采用互斥分类，避免占比超过 100%。
