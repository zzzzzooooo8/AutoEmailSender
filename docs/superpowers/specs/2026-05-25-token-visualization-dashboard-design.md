# Token 可视化统计面板设计规格

## 背景

当前项目已经有 Token 消耗记录中心，能够查看功能级 Token 记录、汇总数值和基础趋势图。下一步希望参考 `Wei-Shaw/sub2api` 的统计面板体验，在统计面板中增加一个更完整的 Token 可视化区域，让用户能够快速回答三个问题：

- 当前时间范围内一共消耗了多少 Token？
- Token 消耗在什么时间发生了变化或异常峰值？
- Token 主要由哪些功能、模型和具体任务产生？

本规格只定义可视化面板的产品结构、布局、交互和数据需求，不包含具体实现计划。

## 目标

- 提供一个清晰、紧凑、可扫描的 Token 统计工作台。
- 支持按时间范围筛选，图表随时间范围变化而刷新。
- 提供高质量鼠标悬停体验，帮助用户查看单个时间桶的完整 Token 明细。
- 通过功能分布、模型排行和最近记录，让用户从趋势图快速定位消耗来源。
- 保持与现有统计面板、卡片、按钮和中文文案风格一致。

## 非目标

- 不在第一版提供复杂成本预测、预算告警或自动优化建议。
- 不引入账户级、用户级多租户分析，除非现有数据已自然支持。
- 不重做已有 Token 记录中心的分页和筛选逻辑，只补强可视化表达。
- 不强制切换到第三方图表库；可以优先复用当前自绘图表能力。

## 组件设计

### 1. 顶部控制区

顶部控制区包含标题、时间范围筛选和刷新按钮。

时间范围筛选支持：

- 最近 6 小时
- 最近 24 小时
- 最近 7 天
- 最近 30 天
- 自定义范围

当用户切换时间范围时，所有统计卡片、趋势图、分布图、排行和最近记录都应基于同一时间范围刷新。自定义范围使用开始时间和结束时间，提交后切换到自定义状态。

### 2. 总览指标卡

顶部展示 4 张指标卡：

- 总 Token
- 输入 Token
- 输出 Token
- 缓存命中 Token / 缓存占比

每张卡显示当前时间范围内的汇总值。若后端可以提供上一周期对比，后续可追加增减百分比；第一版不强制要求。

### 3. Token 趋势图

趋势图是面板的主视觉区域，占据整行。

推荐图形为堆叠柱状图：

- 输入 Token 使用主色。
- 输出 Token 使用浅主色或互补色。
- 缓存命中 Token 使用第三色。
- 每个柱代表一个时间桶。

时间桶粒度规则：

- 最近 6 小时：按小时或更细粒度展示，具体取决于现有数据聚合能力。
- 最近 24 小时：按小时展示。
- 最近 7 天、最近 30 天：按天展示。
- 自定义范围：1 天以内按小时，超过 1 天按天。

悬停效果：

- 鼠标进入某个时间桶时，高亮整列背景。
- tooltip 固定显示在鼠标附近，并做视口边界避让。
- tooltip 展示时间、输入 Token、输出 Token、缓存命中 Token、总 Token。
- tooltip 内容应按同一时间桶聚合，不要求用户精确点中柱体。
- 键盘 focus 某个时间桶时，也应触发等价的高亮状态，保证基本可访问性。

### 4. 功能消耗分布

功能消耗分布用于回答“Token 花在哪类功能上”。

第一版统计维度：

- 智能爬取
- 匹配分析
- AI 草稿
- 其他

推荐展示为环形图 + 右侧列表。列表展示功能名称、Token 数、百分比。数据按 Token 总量降序排列。

### 5. 模型消耗排行

模型消耗排行用于回答“哪个模型消耗最多”。

推荐展示为横向条形图，因为模型名称可能较长，横向条更容易比较大小。每行展示：

- 模型名称
- Token 总数
- 占比或相对条形长度

数据按 Token 总量降序排列。第一版展示前 5 个模型，其余模型可合并为“其他”。

### 6. 最近 Token 消耗记录

最近记录用于从图表回到具体任务。

表格字段建议：

- 时间
- 功能
- 标题或任务名称
- 模型
- 输入 Token
- 输出 Token
- 缓存命中 Token
- 总 Token

默认展示最近 5 条或 8 条记录。若用户在趋势图 hover 或点击某个时间桶，后续可以支持将记录列表筛到该时间段；第一版可先不实现点击联动。

## 数据需求

面板需要一个统一的 Token 可视化数据结构，至少包含：

- 当前时间范围的汇总：输入、输出、缓存、总 Token、记录数。
- 趋势 buckets：时间桶开始时间、展示标签、输入、输出、缓存、总 Token。
- 功能分布：功能类型、功能中文名、Token 总数、占比。
- 模型排行：模型名、Token 总数、占比。
- 最近记录：记录 id、时间、功能、标题、模型、输入、输出、缓存、总 Token。

请求参数至少包含：

- `preset`：快捷时间范围。
- `start_at`、`end_at`：自定义时间范围。
- `granularity`：小时或天，可由前端根据时间范围推导，也可由后端返回最终值。

## 状态与错误处理

- 首次加载时显示图表区域骨架或加载态。
- 切换时间范围时保留旧数据，并在图表区域显示轻量刷新态。
- 无数据时展示空状态：“当前时间范围暂无 Token 消耗数据”。
- 加载失败时展示错误信息和重试按钮。
- 快速切换时间范围时，旧请求结果不能覆盖新请求结果。

## 响应式要求

- 桌面端总览卡 4 列展示。
- 中等宽度下总览卡可变为 2 列。
- 移动端所有组件单列展示。
- 趋势图在窄屏下允许横向滚动，避免柱宽被压到不可读。
- 表格在移动端可以隐藏次要列，优先保留时间、功能、模型和总 Token。

## 可访问性要求

- 时间范围按钮需要清晰的选中态。
- 图表时间桶应可通过键盘 focus。
- tooltip 内容不应只依赖颜色表达。
- 图表颜色需要有足够区分度，输入、输出、缓存三类不能使用过于接近的色值。
- 图表按钮和刷新按钮需要可读的 `aria-label` 或文本。

## 验收标准

- 用户可以通过快捷时间范围切换趋势图，图形和汇总数值同步变化。
- 用户可以通过自定义时间范围筛选图表。
- 鼠标悬停趋势图时，当前时间桶高亮，并展示完整 Token 明细。
- 面板可以展示功能消耗分布、模型消耗排行和最近消耗记录。
- 无数据、加载中、刷新中、加载失败四种状态都有明确 UI。
- 桌面端和移动端布局不重叠、不挤压，主要文字可读。

## 规格自检

- 本规格不包含未完成条目。
- 布局、组件和数据需求一致，均围绕同一时间范围刷新。
- 范围聚焦于 Token 可视化面板，未包含预算告警、成本预测等额外功能。
- 自定义范围和快捷范围的关系已明确：自定义使用 `start_at`、`end_at`，快捷范围使用 `preset`。
