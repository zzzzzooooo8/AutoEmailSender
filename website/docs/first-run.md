# 首次配置

第一次使用时，先完成基础配置，再开始抓导师和创建任务。应用首页会根据当前状态显示「首次上手引导」，引导顺序和下面一致。

## 1. 创建发件身份

进入「个人页」，创建一套发件身份。发件身份代表一个邮箱和一组发信账号配置。

建议填写：

- **身份名称：** 用来区分不同邮箱或不同申请方向。
- **发件人姓名：** 邮件中显示给导师的姓名。
- **邮箱地址：** 实际用于发送邮件的邮箱。
- **SMTP：** 用于发送邮件。
- **IMAP：** 用于检测回复。

SMTP 和 IMAP 一般不是邮箱登录密码，而是邮箱服务商提供的客户端授权信息。通常需要先在邮箱的「设置」「账户安全」「客户端授权」或类似页面里开启 SMTP/IMAP，再生成授权码或应用专用密码。

保存后，先点击 SMTP 和 IMAP 测试按钮。两项都通过后，再进入下一步。

## 2. 配置模型

在「个人页」的模型配置里添加一套 OpenAI 兼容模型配置。

OpenAI 兼容接口的意思是：只要模型服务提供 `API Base URL`、`API Key` 和 `模型名称` 这三项，就可以按同样方式接入。

需要填写：

- **配置名称：** 例如「主力模型」。
- **API Base URL：** OpenAI 兼容接口地址。
- **API Key：** 模型服务的密钥。
- **模型名称：** 实际调用的模型名。

如果你还没有选定模型服务，可以先用 [DeepSeek](https://platform.deepseek.com/)。当前比较适合作为默认起点的是 `deepseek-v4-flash`，API Base URL 填 `https://api.deepseek.com`。

保存后，可以先获取模型列表，再测试模型。测试通过后，把这套模型设为当前使用或默认模型。

## 3. 准备材料和邮件模板

在「个人页」继续准备材料和模板。

材料用于匹配和写信，建议至少上传一份主材料，例如：

- 简历
- 研究计划
- 个人陈述
- 成绩或项目说明

默认邮件模板用于生成初始草稿。模板可以包含占位符，例如导师姓名、导师邮箱、学校、学院、研究方向、发件人姓名等。模板不需要一次写到完美，后续在工作区仍然可以逐封审核和修改。

## 4. 发送测试邮件

在「个人页」完成身份、模板和模型配置后，进入「测试写信页」。

测试写信会给当前身份邮箱发送一封测试邮件，用来确认：

- SMTP 能正常发送。
- 模板渲染正常。
- 附件和材料没有异常。
- 模型生成结果可用。

测试邮件不会写入导师任务。

## 5. 补充导师

进入「导师管理」页，准备导师数据。可以使用以下方式：

- 从学校官网发起智能抓取。
- 导入已有导师信息。
- 手动补充或编辑导师资料。

导师信息越完整，后续匹配分析越有参考价值。研究方向、近期论文、邮箱、学校和学院是比较关键的字段。

## 6. 创建首个任务

回到首页，筛选导师并选择目标导师。

如果导师信息里有研究方向或近期论文，可以先计算匹配度。选择导师后创建任务，在工作区审核匹配结果和邮件草稿。

创建任务时再选择发送方式：

- **立即发送：** 任务创建后进入发送流程。
- **定时发送：** 选择发送日期、发送时间窗口和窗口内发送数量，系统会在指定窗口内按数量发送。
