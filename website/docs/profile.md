# 个人配置

个人页集中管理发件身份、申请材料、邮件模板、模型配置和一些辅助设置。第一次使用时，大部分基础准备工作都在这里完成。

## 发件身份

发件身份用于控制邮件从哪个邮箱发出，以及后续如何检测回复。

主要配置项：

- **发件人姓名和邮箱：** 导师收到邮件时看到的发件信息。
- **SMTP 主机、端口和密码：** 用于发送邮件。
- **IMAP 主机、端口和密码：** 用于检测导师回复。

保存身份后，建议立即测试 SMTP 和 IMAP。测试失败时，优先检查授权码、服务器地址、端口和邮箱服务商的第三方客户端设置。

发送日期、时间窗口和窗口内发送数量在创建任务时设置，不属于发件身份本身。

## 材料与模板

材料和模板会影响匹配分析和草稿生成。

主材料通常选择简历、研究计划或个人陈述。系统会把主材料作为匹配和写信的重要参考。

默认邮件模板包含主题和正文。正文可以使用占位符，例如：

- `{{name}}`：导师姓名
- `{{email}}`：导师邮箱
- `{{title}}`：导师职称
- `{{university}}`：导师学校
- `{{school}}`：导师学院
- `{{research_direction}}`：导师研究方向
- `{{sender_name}}`：你的发件人姓名
- `{{sender_email}}`：你的发件邮箱

模板用于生成初稿，不代表最终邮件。发送前仍然需要在工作区逐封审核。

## 模型配置

模型配置用于匹配分析和生成套磁邮件。系统使用 OpenAI 兼容接口，因此可以配置兼容 OpenAI API 格式的模型服务。

建议流程：

1. 填写 API Base URL、API Key 和模型名称。
2. 保存模型配置。
3. 获取模型列表，确认接口可访问。
4. 测试模型，确认模型可以正常返回结果。
5. 将常用配置设为默认模型。

## 测试写信

完成发件身份、材料、模板和模型配置后，进入测试写信页给自己发送一封测试邮件。测试通过后，再开始批量联系导师。

测试写信只用于检查配置，不会创建导师任务。
